# Visual Benchmark — PersonalOps

## Objetivo

Mapear padrões visuais usados por apps de personal trainer, apps de musculação e bibliotecas de exercícios para definir uma linguagem visual própria, didática e escalável.

## Direções visuais possíveis

### Vídeo real

Vantagem: humano e claro.

Riscos: custo, produção contínua, direito de imagem, diversidade corporal, licenciamento e peso de mídia.

### GIF/animação curta

Vantagem: leve, didático, repetível e rápido.

Riscos: licença e consistência visual.

### Boneco neutro

Vantagem: evita direito de imagem de pessoa real, permite padronização, reduz ruído visual e facilita mostrar músculo-alvo.

Riscos: pode parecer genérico ou infantil se mal executado.

### Modelo anatômico/3D

Vantagem: excelente para ensino, biomecânica e músculo trabalhado.

Riscos: custo, complexidade, performance e licenciamento.

## Direção inicial recomendada

Começar com biblioteca licenciada/API e evoluir para sistema visual próprio com boneco neutro, animações curtas e indicação de grupo muscular.
